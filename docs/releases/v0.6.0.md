**Released:** 2026-06-01 · **Tag:** [`v0.6.0`](https://github.com/abilityai/trinity/releases/tag/v0.6.0) · **PR:** [#998](https://github.com/abilityai/trinity/pull/998)

A large reliability + security release. Headline themes: execution/orchestration reliability, security hardening, the stateful Session tab, experimental Voice Workspace, data-retention policies, and the **public-repo cutover of `.claude` and `src/backend/enterprise` to private submodules** (#775, #847 — OSS build verified by the `build-without-submodule` CI gate).

This document lists **every issue** shipped in the release (all 117 `status-in-dev` issues closed on merge), grouped by type.

## Upgrade Notes
- **Submodule cutover:** `.claude` and `src/backend/enterprise` are now **private submodules**. Public OSS clones cannot resolve them — this is expected; the backend boots in OSS-only mode without them (guarded by the `build-without-submodule` CI job). Developers with access run `git submodule update --init --recursive`.
- **Non-root containers (#874):** existing deployments must re-own their data path / `agent-configs` volume — see [docs/migrations/NON_ROOT_CONTAINERS_2026-05.md](../migrations/NON_ROOT_CONTAINERS_2026-05.md).
- **Redis auth (#589):** `REDIS_PASSWORD` + `REDIS_BACKEND_PASSWORD` are now mandatory — see [docs/migrations/REDIS_AUTH.md](../migrations/REDIS_AUTH.md).
- Verify the running build: `curl -s http://localhost:8000/api/version` → `{"version":"0.6.0", ...}`.
