# Trinity v0.6.0 release announcement

Sent to Discord, Slack, Telegram, and the Trinity X account. Long version below (Discord markdown); platform variants used native formatting (Slack mrkdwn, Telegram HTML). Private submodules omitted per operator instruction.

## Long version (Discord / Slack / Telegram)

🚀 **Trinity v0.6.0 is out**

A major reliability + security release — **264 commits, 117 issues closed, 10 contributors**.

**⚡ Reliability**
• Agent-authoritative execution state machine with single-writer projection
• Hardened orchestration, capacity, and circuit-breaker recovery + many long-running-task fixes
• Continuous canary invariant harness — 7 new orchestration checks

**🔒 Security**
• Redis auth now mandatory (`REDIS_PASSWORD` + `REDIS_BACKEND_PASSWORD`)
• Slack/Telegram bot tokens encrypted at rest
• WebSocket JWT moved out of the URL (ticket-based auth)
• Non-root containers across all production images
• Guardrail-bypass RCE closed + Dependabot high/critical remediation

**✨ New**
• **Session tab** — `--resume`-default chat that keeps full working memory across turns
• **Voice Workspace** (experimental) — agent-scoped voice + canvas with Trinity MCP access, admin-gated
• Soft-delete + configurable data-retention/pruning policies
• Default chat timeout 15m → 60m, platform default model setting, A2A Agent Cards

**Upgrade:** set `REDIS_PASSWORD` + `REDIS_BACKEND_PASSWORD`; re-own the data path for non-root containers. Verify: `curl -s localhost:8000/api/version` → `0.6.0`

Full notes → https://github.com/abilityai/trinity/releases/tag/v0.6.0
