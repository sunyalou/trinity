RFC: Agent Validation Spec — We've drafted a comprehensive list of checks for validating Trinity-compatible agents (issue #668). Before we build the API and MCP tool, we want your input on the spec itself.

📄 https://github.com/abilityai/trinity/blob/dev/docs/agent-validation-spec.md

56 checks across 10 categories — file structure, security, `template.yaml` quality, `CLAUDE.md` quality, skill/playbook construction, autonomy design, cross-file consistency, and more. Checks are split between static (file/pattern analysis) and AI-evaluated (logical consistency, skill coherence, approval gate detection).

Key questions we'd love feedback on:
— Are there important checks missing?
— Any checks that seem too strict or too lenient?
— Are the HARD vs SOFT vs INFO severity assignments right?

Leave a comment on #668 or reply here.
