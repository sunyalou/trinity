### LOW / INFORMATIONAL

**L1 — Operator-supplied MCP `server_name` echoed into advisory warnings (terminal-escape, theoretical). — ✅ RESOLVED 2026-05-29.**
`collect_mcp_credential_warnings()` (`template_service.py`) built
`f"MCP server '{server_name}' references ${{{var}}} ..."`, where `server_name`
is an arbitrary JSON object key from the deployer's own
`.mcp.json.template`. `var` is regex-constrained to `[A-Z][A-Z0-9_]*` (safe),
but `server_name` is unconstrained, so a crafted template could embed control
characters / ANSI escapes that surface in the operator's terminal when
`/trinity:onboard` renders the returned `warnings[]`.
- **Why low**: deploy-local is an authenticated, `creator`-gated action and the
  archive is operator-controlled (self-inflicted, not cross-tenant). The string
  is `JSON.stringify`'d by the MCP `deploy_local_agent` tool, so escapes are
  ``-encoded in transit and only manifest if the consumer renders raw.
- **Fix**: added `_sanitize_for_warning()` — strips non-printable characters
  (`"".join(ch for ch in text if ch.isprintable())`, which drops ESC, CR, LF and
  C0/C1 controls) and bounds length to 80 chars + `"..."`. Applied to
  `server_name` before interpolation; `var` is left as-is (already regex-clamped).
- **Coverage**: TDD — two new unit tests in
  `tests/unit/test_deploy_local_validation.py`
  (`test_malicious_server_name_control_chars_stripped`,
  `test_overlong_server_name_truncated`); both watched fail (raw ESC/newline +
  unbounded 500-char name) before the fix, green after. Full file: 13/13 pass.

## Positive observations (security-improving changes)

- **No credential-value leakage.** The advisory path reads the post-merge
  `.env` (which holds real secret values) only through
  `extract_credentials_from_env_example()`, which returns **variable names
  only** (`line.split('=')[0]`, regex-filtered) — values are never read into
  the warning strings. Confirmed by source inspection. Logs record only
  `len(warnings)`, never content.
- **500→400 hardening.** `is_trinity_compatible()` now decodes `CLAUDE.md` with
  `encoding="utf-8"` inside `try/except UnicodeDecodeError/OSError`, converting
  a binary/unreadable file from an unhandled crash (generic 500) into a clean
  `NOT_TRINITY_COMPATIBLE` 400. Reduces error-path information exposure and
  removes a malformed-input crash vector. Verified by
  `test_binary_claude_md_rejected_without_raising`.
- **Fail-fast, no silent fallback.** `DEPLOYED_TEMPLATES_DIR_UNWRITABLE` /
  `WORKSPACE_PREPOP_FAILED` surface as explicit 500s rather than silently
  deploying an empty agent — aligns with the project's no-silent-fallback rule.
- **No new dependencies** (no `requirements.txt` / `package.json` changes).
- **Auth surface unchanged** — the `/api/agents/deploy-local` handler, its
  role gate, and the size/file-count/path-traversal archive guards are not
  touched by this diff.
- **Read bounds.** `CLAUDE.md` is read whole via `read_text()`, but the file
  is bounded by the pre-existing `ARCHIVE_TOO_LARGE` (50 MB) and
  `TOO_MANY_FILES` (1000) guards applied before this function runs — no new DoS
  surface.
- **Pydantic mutable default** (`warnings: List[str] = []`) is safe: Pydantic
  deep-copies field defaults per instance, so this is not the classic
  shared-mutable-default bug.
