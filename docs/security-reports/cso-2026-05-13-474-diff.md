### LOW

**L1 — Empty `uv.lock` committed to working tree**
`uv.lock` is new and contains only metadata (`version`, `revision`,
`requires-python`) — zero `[[package]]` entries. Not gitignored.
Committing an empty lockfile is not a security risk on its own, but it
will create noise in real lockfile audits going forward (every future
dependency addition will show up as "new in lockfile" rather than
"changed"). Either populate it via `uv sync` or remove the stub before
commit.

### INFORMATIONAL

**I1 — Error detail string leaks transport exception class in
`AgentConnectionDroppedError`** *(Auth Boundaries)*
`agent_client.py:653` raises:
```
f"Connection to agent {self.agent_name} dropped mid-flight: "
f"{type(e).__name__}: {e}"
```
This mirrors the existing `AgentNotReachableError` pattern (line 617),
so it is not a regression — internal hostnames and httpx exception
detail were already surfaceable to API callers via the parent class.
Worth noting for the eventual error-sanitization sweep but not
actionable in this diff.

**I2 — 502 vs 503 choice is defense-in-depth, verified by test**
*(Platform Patterns)*
Both production sites (`headless_executor.py:814-830`,
`gemini_runtime.py:665-675`) deliberately raise HTTP 502 instead of
503 on pipe-drop. `task_execution_service.py:622-642` (SUB-003)
auto-switches the subscription on `agent_status_code == 503 OR
is_auth_failure(error_msg)`. The pipe-close detail string
`"Agent subprocess closed before task could complete"` was
verified by `test_is_auth_failure_rejects_pipe_close_detail` to
NOT match any `AUTH_INDICATORS` substring, and
`test_502_pipe_close_does_not_trigger_subscription_switch` pins the
behavior. This removes a pre-existing minor abuse vector (an attacker
who could induce a pipe drop could trigger a subscription auto-switch
on each event). Net security improvement, not a regression.

## Recommendation

**CLEAR** — proceed with commit.

The diff is a defensive-only refactor: it splits transport-level
exceptions into client-side drops (no circuit hit, no subscription
switch) vs. real liveness signals (circuit hit). All three test files
explicitly guard against creating evasion paths. The 502-vs-503 choice
is documented and pinned by tests against the SUB-003 auth-switch
conditional. No secrets, no new dependencies, no injection surface, no
new auth boundary changes.
