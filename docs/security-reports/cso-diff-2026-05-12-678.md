The #678 patch adds three new attack surfaces and each is closed:

1. **Structured error body crosses the agent→backend trust boundary** —
   `_classify_empty_result` now returns a dict containing partial
   `ExecutionMetadata`. The metadata's `error_message` field is
   populated directly from Claude stdout (`stream_parser.py:281`) and
   can carry credentials. Sanitization fires **twice**: agent-server
   side in `error_classifier.py` (`sanitize_dict` on the dict,
   `sanitize_text` on the message) and again on backend ingest in
   `task_execution_service.py` and `routers/chat.py`
   (`sanitize_dict(partial_metadata)` before any DB write or audit
   log). Defense-in-depth is correct.

2. **Auto-retry economics** — gated on `num_turns < 5` AND clean reader-race
   signature AND `parse_failure_count == 0`; capped to one retry
   (`retry_count = 1`, no loop); retry timeout clamped to
   `min(effective_timeout, 300s)` so a 30-min task can't burn another
   30 min. Failed-attempt cost is rolled into the terminal write
   (`previous_attempt_cost` accumulator) so spend tracking reflects
   actual burn rather than the retry slice alone. No double-charge
   pathway.

3. **JSONL auto-persistence (>600s)** — long headless tasks now write
   the full conversation to `~/.claude/projects/-home-developer/<uuid>.jsonl`
   by default. Disk footprint is bounded by the existing 6h
   `SessionCleanupService` sweep (docstring updated). No new code path
   reads this file with attacker-influenced input — `session_id`
   originates from Claude's own stream-json output inside the agent
   container, never from a request body or env-var.

## Finding #1 — JSONL path construction trusts subprocess output — **RESOLVED**

- **Severity:** LOW (informational, pre-existing pattern; hardened in this commit)
- **Confidence:** 7/10
- **Status:** RESOLVED
- **Phase:** A05 Security Misconfiguration
- **Category:** Platform Patterns
- **File:** `docker/base-image/agent_server/services/jsonl_recovery.py:60-99`

**Original description.** The shared snapshot reader built the JSONL
path via `Path(f"{_JSONL_PROJECTS_DIR}/{session_id}.jsonl")`. `session_id`
comes from `metadata.session_id`, which is populated by `stream_parser.py`
from Claude Code's own stream-json output. The trust boundary was
implicit ("Claude's stdout is trusted inside the agent container") and
a malformed session_id from a corrupted stdout line could in theory
direct the reader at a `*.jsonl`-suffixed file outside the projects dir.

**Fix.** `_read_jsonl_records()` now applies two defenses before any
filesystem access:

1. **Shape whitelist** — `_SAFE_SESSION_ID_RE = re.compile(r"^[A-Za-z0-9_-]{1,128}$")`.
   Matches the union of Claude Code UUIDs (8-4-4-4-12 hex), Trinity's
   `uuid.uuid4().hex` derivatives, and Gemini's `uuid.uuid4()[:8]`
   short-IDs. Anything containing `.`, `/`, `\`, whitespace, or null
   bytes returns `(empty, False, "invalid_session_id")` upfront — no
   path object is ever constructed.

2. **Containment check** — after `Path(_JSONL_PROJECTS_DIR).resolve()`
   and `(projects_root / f"{session_id}.jsonl").resolve()`, the result
   must satisfy `is_relative_to(projects_root)`. Belt-and-suspenders
   against symlink swaps on the projects dir itself.

Both rejection paths log a `WARNING` and return a distinct error code
(`invalid_session_id` / `path_outside_projects_dir`) so operators can
distinguish corrupted-stdout events from ordinary file-missing.

**Tests.** `tests/unit/test_jsonl_metadata_recovery.py` adds
`test_read_jsonl_records_rejects_unsafe_session_id` (parametrized over
12 hostile shapes: `../etc/passwd`, `foo/bar`, `with space`, null bytes,
…) and `test_read_jsonl_records_rejects_path_outside_projects_dir`.
24/24 tests in the module pass.
