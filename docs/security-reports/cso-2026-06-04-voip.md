# CSO Audit — VoIP Telephony (VOIP-001, #1056) — `--diff`

**Date**: 2026-06-04 · **Mode**: daily (8/10 gate) · **Scope**: branch `feature/1056-voip-outbound` vs `dev`

## Scope of change
New OSS feature: outbound VoIP calls over Gemini Live. New WebSocket (`/api/voip/voice/{call_id}`),
per-agent Twilio-voice credentials (`voip_bindings`), an outbound call trigger (REST + MCP), and
post-call transcript processing. Feature-flag gated OFF by default.

## Attack surface added
| Surface | Auth | Notes |
|---------|------|-------|
| `GET/PUT/DELETE /api/agents/{name}/voip` | Owner (`OwnedAgentByName`) | Binding CRUD |
| `POST /api/agents/{name}/voip/call` | JWT/MCP (`AuthorizedAgent`) | Rate-limited + daily cap + Idempotency-Key |
| `WS /api/voip/voice/{call_id}` | Call-bound single-use ticket (no JWT) | Twilio Media Streams |
| MCP `call_user` | Agent-scoped key | Proxies the REST trigger |

## Findings

**No CRITICAL or HIGH findings.** Confirmed clean across the diff:

- **Secrets archaeology (P2)** — VERIFIED clean. No keys/tokens in the diff; test fixtures use
  fake tokens (`FAKE-…`, `AC0000…`).
- **A02 Cryptographic** — VERIFIED. Twilio AuthToken stored via AES-256-GCM JSON envelope
  (`CredentialEncryptionService`, Invariant #12); never logged or returned (response model has no
  token field). Re-encrypts with fresh nonce on update.
- **A01 Access control** — VERIFIED. Binding CRUD owner-only; trigger access-gated; the Media
  Streams WS (Twilio can't send a JWT) is gated by a **single-use, call-bound** ticket
  (`scope="voip:{call_id}"`, verified against the URL `call_id`), and the staged intent is consumed
  exactly once via Redis `GETDEL`. A leaked/replayed ticket can't attach to a different call and
  can't be reused.
- **A03 Injection** — VERIFIED. All DB writes parameterized; the dynamic `SET` in
  `update_call_status` joins only hardcoded column fragments (values bound). TwiML URL attribute is
  `quoteattr`-escaped (covered by a unit test). No `subprocess`/`eval`/`exec`/`shell=True` in the
  new code.
- **A04 Insecure design / cost** — VERIFIED. Outbound calls (real PSTN spend) are bounded by a
  per-`(owner, destination)` sliding-window rate limit + a **durable** per-agent daily cap +
  flag-off-default + per-agent own-Twilio credentials. Idempotency-Key prevents double-dial.
- **A09 Logging** — VERIFIED. `voip_call_placed` is audit-logged (to_number + call_id; no token).
- **A10 SSRF** — VERIFIED not exploitable. The only outbound fetch is Twilio Account validation at
  the **fixed** host `api.twilio.com`; `account_sid` (AC + 34 chars) appears only in the path and
  cannot override the authority. No user-controlled host. No media downloads in Phase 1.

## Informational (no action required for Phase 1)

- **Prompt-injection surface via call transcript → main agent**: the callee's spoken words are
  included (framed under `## Call transcript / Caller:`) in the post-call `execute_task` message to
  the main agent. The callee is a third party **chosen by the agent owner** (owner supplies the
  destination number), and this is the platform's existing "process untrusted external content"
  pattern (identical to channel messages / emails fed to agents via `message_router`). Mitigated by
  framing + owner-chosen recipient + the agent's own guardrails. Not a new vulnerability class.
- **Trigger uses `AuthorizedAgent`** (access-based, not owner-only) — deliberate, bounded by the
  abuse controls above. A formal opt-in destination allowlist is the documented Phase 2 hardening.

## Verdict
The diff is safe to land. Credential handling, the new WSS auth surface, and the money-spending
trigger all follow established Trinity patterns with appropriate controls.
